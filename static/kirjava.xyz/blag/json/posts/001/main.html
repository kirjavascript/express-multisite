I first started learning about scaleable vector graphics (SVG) when learning the D3 library. Initially, it was mostly a means to an end and I wasn't aware how useful it could be when creating (needlessly flashy) web effects.

I use Inkscape when I need a fully fledged editor, it's a powerful alternative to adobe illustrator.

Inkscape tends to produce pretty syntactically verbose markup. SVGOMG is a web GUI for SVGO, an SVG minifier. This is useful for way more than just decreasing the filesize, it supports beautification, can rewrite paths to adjust for group transforms or viewBox, combine paths, and much more.

There are various web based tools for SVG, but I haven't found one that I'd prefer to use over Inkscape.

SVG2D3 is a really useful tool to convert from SVG markup to d3 function calls. The specification agnostic nature of D3 makes creating something like this pretty straightforward.
